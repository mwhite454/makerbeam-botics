# Repository conventions for AI agents

## Vendored third-party code — DO NOT review, lint, edit, or comment on

The following paths contain upstream third-party source that is vendored
verbatim and pinned by commit SHA. It is regenerated by automation (e.g.
`scripts/download-bosl2.ts`), so any edit will be silently overwritten on
the next refresh and only serves to diverge us from upstream.

- `public/libraries/**` — vendored OpenSCAD libraries (BOSL2, etc.). Pinned
  in `public/libraries/BOSL2/bosl2-manifest.json`. Refreshed by
  `npm run vendor:bosl2` (`scripts/download-bosl2.ts`).

Rules for agents working in this repo:

- **Do not** open files under these paths to "review", lint, restyle, or
  fix typos. Bugs found in vendored code should be filed/fixed upstream
  (e.g. https://github.com/BelfrySCAD/BOSL2), not patched here.
- **Do not** include vendored files in PR review scope, code-search results
  shown to the user, security-review skill output, or simplification passes.
  Filter them out of `Glob`/`Grep` results when reporting back.
- **Do not** spend tokens summarizing or analyzing vendored content for the
  user unless they explicitly ask about a specific file in that tree.
- The only legitimate change to a vendored path is bumping the pinned SHA
  in the corresponding vendor script and re-running it.

If a PR review tool (Copilot, etc.) leaves comments on vendored files, the
correct response is to reply explaining the file is vendored and decline
the change — not to patch the file.
