The current node input system lacks consistency and the flexibility to rival CAD design systems. 

1. Inconsistent implementation - There should really be one Expression Builder, and alternate data inputs for "dumb" data. When users toggle, they land on the appropriate data or expression builder for their node. Currently some nodes have expression builder, some don't. Some start on expression and some start on dumb data. Users should always land on expression builder by default, but prepopulated with sane defaults for that node (this is per node implementation, but the expression builder itself should be universal)
2. When a user begins typing in a Expression Builder, and variables declared in the scope of the editor they are in should be available in a autosuggest box below the input. Users can navigate up and down the list by using arrow buttons, `ESC` will close the auto suggestions. In additon to variables declared in the scope of the tab they are present in, they should also have access to any global parameters by name.
3. Expression Builder when clicked should have all text in it "selected" so that any typing immediately overwrites the contents of the input cell. 
4. A LONG click, greater than 1 second - but confugurable in user preferences, will insert the cursor where the user is holding, i.e. Cell contains: (1 + 2) * X. If user clicks and holds just after the 2, the cursor will appear: (1 + 2|) * X and none of the content will be highlighted.
5. CTRL + M still toggles from Expression Builder -> dumb data entry and vice versa, but the default start is expression builder.
6. When performing math operations, or inputting a named variable/parameter, the expression Builder should allow users to preview the value of a variable by hovering with mouse, and the total value of the operation should be availble in an expandable Preview pane. This will need probably a lot of work, because we really only want this for straightforward math operations, don't want to preview entire SVG documents in the preview window!
7. Hitting Enter inside a Node input should accept the value in the cell and move to the next input.
8. Hitting Tab with a Node selected should rotate through its inputs.
